## Steps

Use `Steps` and `Step` to display sequential workflows:

<Steps>
  <Step title="Install Dependencies">
    Clone the repository and install project dependencies:

    ```bash
    git clone https://github.com/Coldplay-now/ezdoc.git
    cd ezdoc && pnpm install
    ```
  </Step>
  <Step title="Write Documentation">
    Create a new `.mdx` file under the `docs/zh/` directory, and add frontmatter and content.
  </Step>
  <Step title="Local Preview">
    Run `pnpm dev` to start the development server and preview the results in your browser.
  </Step>
  <Step title="Build and Deploy">
    Run `pnpm build` to generate static files, then deploy to GitHub Pages or any static hosting service.
  </Step>
</Steps>

### Syntax

```mdx
<Steps>
  <Step title="First Step">
    Step content, supports **Markdown** and code blocks.
  </Step>
  <Step title="Second Step">
    More content...
  </Step>
</Steps>
```

## Accordion

Use `Accordion` and `AccordionItem` to create collapsible content sections:

<Accordion>
  <AccordionItem title="What Markdown syntax does ezdoc support?">
    ezdoc fully supports GitHub Flavored Markdown (GFM), including tables, task lists, strikethrough, autolinks, and more. It also supports math formulas (KaTeX) and Mermaid diagrams.
  </AccordionItem>
  <AccordionItem title="How do I add a new documentation page?">
    Create an `.mdx` file under the `docs/zh/` directory, then add the corresponding path to the navigation section in `docs/zh/docs.json`.
  </AccordionItem>
  <AccordionItem title="Is multi-language supported?">
    Yes. Configure `i18n.locales` in `ezdoc.config.ts`, then create a separate documentation directory (e.g., `docs/en/`) and navigation file for each language.
  </AccordionItem>
</Accordion>

### Syntax

```mdx
<Accordion>
  <AccordionItem title="Question title">
    Answer content, supports **Markdown** formatting.
  </AccordionItem>
</Accordion>
```

## FileTree

Use `FileTree` to display project directory structures:

<FileTree json='["docs/", ["zh/", ["docs.json", "getting-started.mdx", "components.mdx", "guide/", ["configuration.mdx", "deployment.mdx", "extended-components.mdx"]], "en/", ["docs.json", "getting-started.mdx"]], "ezdoc.config.ts", "package.json"]' />

### Syntax

Pass a JSON-formatted array string via the `json` prop. Folder names end with `/` and are followed by a sub-array representing their contents. File names do not end with `/`.

```mdx
<FileTree json='["src/", ["components/", ["Button.tsx", "Header.tsx"], "lib/", ["utils.ts"]], "package.json"]' />
```

## ImageZoom

Use `ImageZoom` to enable click-to-zoom for images:

<ImageZoom src="https://github.githubassets.com/assets/github-logo-55c5b2b2feaa.png" alt="GitHub Logo" />

Click the image above to view it in full screen. Press `ESC` or click the background to close.

### Syntax

```mdx
<ImageZoom src="/images/screenshot.png" alt="Screenshot description" />
```

| Prop | Description | Default |
|------|-------------|---------|
| `src` | Image URL | Required |
| `alt` | Alt text | Optional |
| `width` | Image width | Optional |
| `height` | Image height | Optional |

## Tooltip

Use `Tooltip` to add hover hints to terms or text:

ezdoc uses the <Tooltip content="A format that combines Markdown with JSX components">MDX</Tooltip> format for writing documentation, supports <Tooltip content="GitHub Flavored Markdown, an extended Markdown syntax by GitHub">GFM</Tooltip> syntax, and provides full-text search powered by <Tooltip content="A static search engine that generates indexes at build time">Pagefind</Tooltip>.

### Syntax

```mdx
<Tooltip content="Tooltip content">Annotated text</Tooltip>
```

Hover over text with a dashed underline to see the tooltip.
